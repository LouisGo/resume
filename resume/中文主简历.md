### **TypeX PC & TypeX AI 应用架构建设**

TypeX PC 是面向高频 IM 协作场景的桌面端产品，核心复杂度集中在消息列表、消息渲染、多端 SDK 对接、AI 能力集成和高可靠交互体验。随着大模型能力快速发展，项目逐步从传统 IM 客户端向 AI Native 工作台演进。

作为**前端架构师 / AI 应用落地核心开发者**，负责核心前端架构升级和 AI 应用落地，重点推进 AI runtime、IM Message List、Message Render、全局 actions、日志监控和工程质量体系建设。

- **AI runtime 架构落地**：参与 TypeX AI 从 0 到 1 建设，将分散在聊天、翻译、CLI、Plugin 等入口中的 AI 能力收敛为「入口层 / 编排层 / 接入层」三层 runtime，统一上下文构建、流式响应、会话状态、基础记忆、安全确认、多模型 provider、Agent Provider、Tool Use、MCP / OpenClaw 和内部 tools，对接 **【8】家国内外大模型厂商**，沉淀 **【N】类内部 tools**，覆盖 **【N】个核心业务场景**。
- **IM Message List 架构演进**：主导消息列表多阶段重构，在不阻塞业务迭代的前提下完成从虚拟列表优化、应用级数据管理到 XMessageList 消息锚点滚动容器的渐进式迁移；通过新旧链路并行、兼容适配和回滚预案 **【待补迁移证据】**，解决滚动卡顿、动态高度跳变、频繁重渲染、数据一致性和复杂场景恢复问题，使核心会话场景滚动流畅度提升至 **【55-60 FPS】**，重渲染次数降低 **【60%+】**。
- **Message Render 分层架构**：核心负责 Message Render 建设，将 SDK 原始消息收敛为「领域模型 / 类型注册表 / 能力声明 / 布局策略 / 渲染插件」的分层体系，覆盖十几种消息类型和多种业务场景；通过控制反转、策略分发和场景化 hooks，将渲染、布局、交互、权限和上下文解耦，解决历史组件层层透传 props、分支膨胀和新增类型牵动多处的问题，使消息扩展从「改业务组件」转为补充独立能力单元。
- **全局 actions 设计**：设计统一 actions 体系，将复制、转发、引用、多选、跳转等业务动作抽象为可注册、可组合、可复用的 action descriptor，由统一执行层处理上下文注入、可用性判断、权限约束和批量语义，使右键菜单、快捷操作、多选工具栏和 AI 调用入口仅消费同一动作契约。
- **富文本输入与渲染链路**：基于 Tiptap / ProseMirror 打磨 IM 富文本 composer，处理 mention、粘贴、草稿、全屏编辑和快捷输入等高频场景；在消息渲染侧沉淀剪贴板策略、HTML 清洗白名单和富文本解析规则，统一输入态、存储态和展示态协议，降低富文本格式丢失、复制粘贴错乱、XSS 风险和特殊消息渲染异常。
- **稳定性与工程效率治理**：围绕消息列表、消息渲染和 AI runtime 建立可回归的质量体系，将关键 transform、capability、layout、剪贴板、滚动恢复、provider 兼容和运行时错误处理纳入测试与日志诊断；同步优化本地校验和开发链路，让高风险重构能按小步提交、快速验证、可回滚的方式推进，减少复杂问题定位和回归成本。
