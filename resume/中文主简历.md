## 主要项目经历

### **TypeX PC & TypeX AI 复杂客户端架构建设**

TypeX PC 是面向高频 IM 协作场景的桌面端产品，核心复杂度集中在消息列表、消息渲染、富文本输入、多端 SDK 对接、AI 能力集成和高可靠交互体验。随着大模型能力进入产品主流程，项目逐步从传统 IM 客户端演进为可承载 AI 工作流的复杂桌面端。

作为**前端架构师 / AI 应用落地核心开发者**，负责在不阻塞业务迭代的前提下推进核心链路重构：先稳定高频 IM 交互底座，再抽象消息渲染与动作体系，最后承接 AI runtime 与产品内部能力的接入。

- **AI runtime 与产品能力接入**：参与 TypeX AI 从 0 到 1 建设，将聊天、翻译、CLI、Plugin 等分散入口中的 AI 能力收敛为可复用 runtime，统一处理上下文构建、流式状态、会话记忆、安全确认、多模型 provider、Agent Provider、Tool Use、MCP / OpenClaw 和内部 tools 接入。设计重点从“调用模型”前移到“产品入口、工具边界、确认机制、失败恢复和可观测性”，使 AI 能力可以复用在多个真实业务场景中。
- **IM Message List 高频交互治理**：主导消息列表多阶段演进，从虚拟列表性能优化、应用级数据管理，到 XMessageList 消息锚点滚动容器的渐进式迁移；通过新旧链路并行、兼容适配和回滚预案，治理滚动卡顿、动态高度跳变、频繁重渲染、数据一致性和复杂场景恢复问题，使核心会话场景滚动流畅度、状态恢复和问题定位能力持续提升。
- **Message Render 与 actions 分层架构**：核心负责 Message Render 建设，将 SDK 原始消息收敛为「领域模型 / 类型注册表 / 能力声明 / 布局策略 / 渲染插件」的分层体系，并将复制、转发、引用、多选、跳转、AI 调用等动作统一为可注册、可组合的 action descriptor。通过控制反转、策略分发和场景化 hooks，将渲染、布局、交互、权限和上下文解耦，使新增消息类型和业务动作从“改主流程”转为补充独立能力单元。
- **富文本链路与工程质量治理**：基于 Tiptap / ProseMirror 打磨 IM 富文本 composer，处理 mention、粘贴、草稿、全屏编辑、快捷输入、HTML 清洗和剪贴板策略等高频细节；围绕消息列表、消息渲染和 AI runtime 建立可回归的测试与日志诊断链路，将关键 transform、capability、layout、滚动恢复、provider 兼容和运行时错误处理纳入日常验证，让高风险改造按小步提交、快速验证、可回滚的方式推进。

**入职早期即凭借虚拟列表和消息列表优化成果获得提前转正；后续因消息渲染、AI 能力落地和核心架构演进等关键贡献，获得年度最高等级 bonus。**

- **低代码平台架构设计**：基于 qiankun 微前端和 JSON Schema DSL，完成数据、视图、操作、权限和业务配置的模块化解耦，将复杂平台拆分为可并行交付、可扩展的子应用体系，为后续插件市场和多团队协作提供基础，协调 **5+** 子应用团队并行交付。
- **核心视图引擎设计**：面向低代码平台中最高频的表格视图，设计数据层、视图层、交互层和渲染调度层分离的视图架构，支撑百万级数据下的滚动、拖拽、动态行高、单元格编辑和批量更新等复杂交互。
